<summary><b>⚙️ One-time setup for the metrics above (click to expand)</b></summary>
<br>

The streak stats and activity graph render immediately — no setup needed. The **metrics card**, **isometric calendar**, and **contribution snake** are generated on your own GitHub Actions via [`.github/workflows/metrics.yml`](.github/workflows/metrics.yml) and [`.github/workflows/snake.yml`](.github/workflows/snake.yml), using [lowlighter/metrics](https://github.com/lowlighter/metrics) and [Platane/snk](https://github.com/Platane/snk). Running them on your own repo means they won't go down or get rate-limited the way shared free-tier widgets (e.g. `github-readme-stats.vercel.app`, `github-profile-trophy.vercel.app`) sometimes do — both were sitewide-down at the time this README was written, which is why those particular widgets were left out.

**Setup steps:**

1. Create a **classic Personal Access Token**: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token, with scopes: `repo`, `read:user`, `read:org`.
2. In this repo → Settings → Secrets and variables → Actions → New repository secret → name it `METRICS_TOKEN` → paste the token.
3. Go to the **Actions** tab and manually run **"Generate Profile Metrics"** once (or wait for the daily 00:00 UTC cron). It commits `github-metrics.svg` and `github-isocalendar.svg` to the repo root.
4. **"Generate Contribution Snake"** runs automatically on push to `main` (and daily) — it pushes the generated SVGs to an `output` branch, which the snake image above reads from. No extra secret needed; it uses the built-in `GITHUB_TOKEN`.
5. Once both workflows have run at least once, every image in this section renders automatically — no further steps needed.

</details>
